## 简介

Qoder-todo是一个基于FastAPI的客户资产分析工具，提供了从数据上传、分析到PDF报告生成的完整工作流程。本文档旨在为该系统建立一套完整的监控与日志管理体系，包括系统资源监控、应用性能监控、业务指标监控、结构化日志管理、错误追踪、APM工具集成以及告警机制设置。

### 日志级别配置

| 日志级别 | 使用场景 | 示例消息 |
|---------|---------|---------|
| DEBUG | 开发调试信息 | "开始处理文件上传" |
| INFO | 重要业务事件 | "任务创建成功: task-123456" |
| WARNING | 可能的问题但不影响功能 | "文件大小接近限制: 95%" |
| ERROR | 错误但可恢复的异常 | "分析失败: OpenAI API错误" |
| CRITICAL | 严重错误影响系统运行 | "PDF生成失败: 字体注册失败" |

### 日志轮转策略

采用基于时间和大小的双重轮转策略：

- **时间轮转**: 每天午夜自动轮转
- **大小轮转**: 单个日志文件达到100MB自动轮转
- **保留策略**: 保留最近7天的日志文件
- **压缩策略**: 旧日志文件自动压缩
- **清理策略**: 清理超过30天的日志文件

## APM工具集成

推荐使用Sentry作为APM工具，实现完整的错误追踪和性能监控。

#### 安装配置

```bash
pip install sentry-sdk
```

#### 基础配置

```python
import sentry_sdk
from sentry_sdk.integrations.fastapi import FastApiIntegration
from sentry_sdk.integrations.logging import LoggingIntegration

sentry_sdk.init(
    dsn="YOUR_SENTRY_DSN",
    integrations=[
        FastApiIntegration(),
        LoggingIntegration(level=logging.INFO, event_level=logging.ERROR)
    ],
    traces_sample_rate=1.0,
    profiles_sample_rate=1.0,
)
```

#### 性能监控配置

```python
# 自动监控API端点性能
@router.middleware("http")
async def performance_monitor(request, call_next):
    start_time = time.time()
    response = await call_next(request)
    duration = time.time() - start_time
    
    # 上报性能指标到Sentry
    sentry_sdk.set_measurement("response_time", duration, "millisecond")
    
    return response
```
